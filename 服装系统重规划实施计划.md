## 第三阶段：统一换装管线

**新建文件：** `publish/scripts/features/npc-auto-dress.js`

### 3.1 管线入口 — `appearancePipeline`

```js
Game.appearancePipeline.apply(state, person, reason)
```

内部流程：
1. 记录当前 reason 和优先级
2. 如果有更高优先级覆盖 → 跳过
3. 否则按 reason 执行对应换装
4. 存储 `person._lastDressReason` 和 `person._lastDressPriority`

### 3.2 优先级表

| 优先级 | 标识 | 说明 |
|--------|------|------|
| 0 | genetics | 基因基础——出生时一次性 bodyType/hairstyle/hairColor |
| 1 | phase-style | 年龄阶段——阶段变化时按 stylePreference 全套 |
| 2 | npc-init | NPC 初始——NPC 生成时偏好+文化全套 |
| 3 | culture | 文化适配——文化切换时可能改为 wafuu 偏好 |
| 4 | career | 职业换装——定期 24-60 月；职业要求 formal 时临时覆盖 |
| 5 | encounter | 遭遇换装——交欢/风俗临时装扮；结束后回退到 4 |
| 6 | specter | 幽诡寄生——强制偏好 school/wafuu；不可被低优先级覆盖 |
| 7 | cosplay | Cos服进化——top/bodyType 锁定角色名 |
| 8 | cradle | 摇篮改造——强制偏好 wafuu |
| 9 | surgery | 整容手术——bodyType 永久变更 |
| ∞ | player-lock | 玩家手动——最高优先，永久锁定至下次手动更换 |

### 3.3 遭遇回退机制

```
遭遇开始:
  pipeline.apply(state, person, 'encounter-start')
    → 保存当前装扮到 person._preEncounterOutfit
    → 应用遭遇装扮

遭遇结束:
  pipeline.apply(state, person, 'encounter-end')
    → 恢复 person._preEncounterOutfit
    → 清除临时记录
```

### 3.4 幽诡寄生覆盖（偏好驱动，不固定单品）

**华夏幽诡寄生：**
```
pipeline.apply(state, person, 'specter-possess')
  → 强制大池子 = school
  → 小池子从 [sailor(60%) jk(25%) blazer(15%)] 随机
  → 由 stylePreference.pickOutfit() 随机全套（每次寄生可能不同）
```

**日本幽诡寄生：**
```
pipeline.apply(state, person, 'specter-possess-jp')
  → 强制大池子 = wafuu
  → 小池子从 [miko(55%) furisode(25%) taisho(15%) casual(5%)] 随机
```

**幽诡亚种微调小池子权重：**

| 幽诡类型 | sailor | jk | blazer |
|---------|--------|-----|--------|
| 怨灵 | 50% | 30% | 20% |
| 魅妖 | 70% | 20% | 10% |
| 淫妖 | 75% | 15% | 10% |
| 恶煞 | 30% | 40% | 30% |
| 梦魇 | 45% | 35% | 20% |
| 情缚 | 60% | 25% | 15% |
| 欲母 | 50% | 20% | 30% |

净化后：不恢复原偏好，保持寄生时的偏好。
